1. OTP EMAIL NOT BEING RECEIVED  [CRITICAL]
──────────────────────────────────────────
Problem:
  - MAIL_USERNAME and MAIL_PASSWORD are read from os.environ.get().
  - On Render, these env vars MUST be manually set in the Render Dashboard
    under Environment → Environment Variables.
  - If they are missing, mail.send() silently returns False and no email
    is dispatched — no crash, but no OTP either.
  - The Gmail App Password (dntfgrfqlsnmvjyz) may also be expired or
    revoked by Google if 2FA settings changed.

Fix Required:
  - Go to render.com → Your Service → Environment
  - Add: MAIL_USERNAME=opercent517@gmail.com
  - Add: MAIL_PASSWORD=dntfgrfqlsnmvjyz
  - Add: MAIL_DEFAULT_SENDER=opercent517@gmail.com
  - Add: MAIL_SERVER=smtp.gmail.com
  - Add: MAIL_PORT=587
  - Add: MAIL_USE_TLS=True
  - Add: MAIL_USE_SSL=False
  - Verify the Gmail App Password is still valid at:
    https://myaccount.google.com/apppasswords

2. DOUBLE APP INITIALIZATION  [CRITICAL]
──────────────────────────────────────────
Problem:
  - app.py now exposes `app = create_app()` at module level.
  - wsgi.py was also calling `app = create_app()` separately.
  - This caused MongoDB and Flask-Mail to be initialized TWICE,
    leading to double connections and potential startup errors.

Fix Applied:
  - wsgi.py now imports the already-created app: `from app import app`

3. HTTP 502 ON REGISTRATION  [HIGH]
──────────────────────────────────────────
Problem:
  - If email sending takes >30s (SMTP timeout), Gunicorn's request
    times out before a response is sent, causing a 502 Bad Gateway.
  - The default Gunicorn worker timeout may not be enough.

Fix Required:
  - Procfile already sets --timeout 120, which should help.
  - Consider sending OTP email asynchronously (e.g. using threading
    or a background task queue like Celery/RQ) in future.

9. OTP EMAIL GOES TO SPAM  [MEDIUM]
──────────────────────────────────────────
Problem:
  - Emails sent from a Gmail account via SMTP App Password frequently
    land in spam folders because the sender domain (gmail.com) has
    no custom domain SPF/DKIM configured for this platform.

Fix Required:
  - Add "Check your spam folder" note on the OTP verify page.
  - Consider using a transactional email service like SendGrid or
    Brevo (free tiers available) for better deliverability.
